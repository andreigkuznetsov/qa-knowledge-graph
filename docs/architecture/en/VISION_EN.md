# QA Knowledge Graph --- Vision 2030

## Vision

> Build the first open knowledge platform for Software Quality
> Engineering that understands the meaning of requirements, business
> rules, scenarios, tests, implementations and their relationships,
> helping engineers make better technical decisions.

## Why this project exists

Modern engineering stores information across many systems:

-   Jira stores work items.
-   Confluence stores documentation.
-   Swagger stores APIs.
-   Git stores code.
-   Test management systems store test cases.
-   CI/CD stores execution results.

These systems answer **what exists**.

They rarely answer **why it exists** or **what will happen if it
changes**.

QA Knowledge Graph introduces a Knowledge Layer that transforms
disconnected artifacts into connected engineering knowledge.

## Mission

Transform engineering artifacts into knowledge and transform knowledge
into explainable decisions.

## Success

A software engineer should be able to ask:

-   What should I test after this change?
-   Which business rules are not protected?
-   Why does this test exist?
-   Which scenarios are affected?

...and receive an explainable engineering answer rather than a list of
files.
