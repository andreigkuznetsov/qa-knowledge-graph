# QA Knowledge Graph --- Видение 2030

## Видение проекта

> Создать первую открытую платформу инженерных знаний для **Software
> Quality Engineering**, которая понимает смысл требований,
> бизнес-правил, сценариев, тестов и технической реализации, помогая
> инженерам принимать более качественные технические решения.

## Почему существует этот проект

Современная разработка хранит информацию в разных системах:

-   Jira --- задачи;
-   Confluence --- документацию;
-   Swagger/OpenAPI --- API;
-   Git --- исходный код;
-   Test Management --- тесты;
-   CI/CD --- результаты выполнения.

Все они хорошо отвечают на вопрос **«Что существует?»**.

Но почти не отвечают на вопросы:

-   Почему это существует?
-   Что изменится после модификации требования?
-   Какие бизнес-правила будут затронуты?
-   Какие тесты необходимо выполнить?

QA Knowledge Graph добавляет **Knowledge Layer**, превращая разрозненные
артефакты в единую систему инженерных знаний.

## Миссия

Преобразовывать инженерные артефакты в знания, а знания --- в объяснимые
инженерные решения.

## Дорожная карта

### Phase 1 --- Foundation

-   Canonical QA Model
-   JSON Schema
-   Semantic Validation
-   Story Analyzer
-   Story Extractor

### Phase 2 --- Knowledge Intelligence

-   Coverage Analysis
-   Impact Analysis
-   Risk Analysis
-   Traceability Engine

### Phase 3 --- Knowledge Graph

-   Graph Database
-   Dependency Explorer
-   Visual Knowledge Graph

### Phase 4 --- AI Architect

-   объяснение решений;
-   анализ рисков;
-   рекомендации по недостающим тестам;
-   анализ влияния изменений.

### Phase 5 --- Enterprise Platform

-   Jira
-   GitHub / GitLab
-   Test Management
-   CI/CD
-   Documentation Systems

## Критерий успеха

Инженер должен иметь возможность спросить:

-   Что необходимо протестировать после этого изменения?
-   Какие бизнес-правила не защищены тестами?
-   Почему существует этот тест?
-   Какие сценарии будут затронуты?

...и получить аргументированный инженерный ответ, а не просто список
файлов.
